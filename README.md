🖥️ Slime OS 操作ガイド

Slime OSは、デスクトップ画面を起点に、各種ツールやフォルダへアクセスして利用するOS風インターフェースです。
1. 基本的な操作スタートメニューの開閉画面左下の「Slime OS」ボタンをクリックすると、メニューが表示・非表示になります。
2. フォルダ・アプリへのアクセスデスクトップやスタートメニューから、以下の場所へ移動できます： 🌿 スライムの森 (Home)💧 水辺の広場⚙️ 設定📅 カレンダー📚 Office (オンライン文書編集)📝 Document (文書管理)📥 Download (ダウンロードファイル管理)🎬 Video / 🖼️ Picture
3. アプリケーションの操作ファイルマネージャー（Document / Download等）各フォルダ内のファイルをクリックすると、ファイル詳細や内容を表示するウィンドウが起動します。
4. ウィンドウはドラッグして移動させたり、右上の「❌」ボタンで閉じたりできます。  テキストエディタ新規作成または既存のテキストファイルを開くと起動します。  保存: 「💾 保存」ボタンで内容をファイルとしてダウンロードできます。  編集: 自由に文字入力ができるほか、「🕒 時刻挿入」で現在のタイムスタンプを追加可能です。  Officeアプリ「📚 Office」内のファイルをクリックすると、オンラインエディタウィンドウが開き、ドキュメントや表計算の閲覧が可能です。
5. ログインについてシステムへのログインIDとパスワードは以下の通りです。ログインID: user  パスワード: password ※このシステムはデスクトップ画面や各フォルダを通じて、直感的にファイル管理や簡単な作業ができるよう設計されています。

6. 開発・検証プロセス報告書
7. 
1. 開発目的
Webアプリケーション「スライムOS」における、ユーザー認証機能およびデータ永続化の実装と動作検証。
2. 取り組んだ内容
環境構築: Java（Eclipse）およびSpring Bootを用いたWebサーバーの立ち上げ。
データベース実装: H2 Databaseを用いた、アプリケーション内でのデータ管理環境の構築。
認証機能の実装: ログイン画面の作成と、Google Chromeブラウザのパスワード管理機能との連携による、安全かつ円滑な認証フローの確立。
動作検証: アプリケーション起動からログイン認証、およびH2コンソールを使用したデータベース接続とテーブル（USERSテーブル等）の確認。
3. 実行時の流れ（操作手順）
システム起動: Eclipse上でプロジェクトを実行し、サーバーを起動させる。
ログイン画面アクセス: ブラウザで http://localhost:8080/start.html にアクセスし、ログイン画面を表示。
認証と保存: ユーザー名とパスワードを入力し、ブラウザの自動保存機能を利用して認証を行う。これにより、ユーザー体験（UX）の向上とセキュリティ確保を確認。
データベース接続確認: http://localhost:8080/h2-console にアクセスし、H2コンソールへログイン。
データ構造検証: コンソール上でデータベース内の「USERS」テーブル等の存在を確認し、データが正しく管理されている状態を検証。
4. 考察・成果
本プロジェクトを通じて、単なる画面表示だけでなく、裏側のデータベース処理（CRUD操作の基盤）までを一貫して実装できた。また、開発環境におけるポート競合の解消や、データベース接続エラーのデバッグなど、実践的なトラブルシューティング能力を養うことができた。ブラウザの認証機能との連携により、ユーザーが日常的に利用するWebサイトと同等の利便性を備えたシステム構築を実現した。
