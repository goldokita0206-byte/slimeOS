# Slime OS - Webデスクトップシステム
## 🎓 職業訓練校 課題提出用
### 8. 開発目的・成果
本プロジェクト「Slime OS」は、Webアプリケーション開発技術の習得を目的とし、**ユーザー認証機能**と**データ永続化**を備えた、OS風のインターフェースを開発・検証する課題です。

### 9. 使用技術・開発環境
- **開発言語・フレームワーク**：Java 17 / Spring Boot 3.4.1
- **開発ツール**：Eclipse IDE
- **データベース**：H2 Database（インメモリ型）
- **フロントエンド**：HTML5 / CSS3 / JavaScript / Tailwind CSS
- **ブラウザ**：Google Chrome
- **バージョン管理**：Git / GitHub
- **機能連携**：ブラウザパスワードマネージャーによる自動保存・安全な認証フロー

### 📖 Slime OS 操作ガイド
Slime OSは、デスクトップ画面を起点に、各種メニューやフォルダへアクセスして利用するOS風インターフェースです。

1. **基本操作**
   スタートメニューは画面左下の「Slime OS」ボタンをクリックして開きます。もう一度クリックするか、画面外をクリックすると閉じます。

2. **メニュー・フォルダ一覧**
   スタートメニューまたはサイドバーから、以下の機能へ移動できます：
   - 🌿 スライムの森（Home）
   - 💧 水辺の広場
   - ⚙️ 設定
   - 📅 カレンダー
   - 📚 Office（オンライン文書編集）
   - 📄 Document（文書管理）
   - 📥 Download（ダウンロード管理）
   - 🎬 Video / 🖼️ Picture

3. **ファイル操作**

5. **ウィンドウ操作**
   ウィンドウはドラッグで移動、右上の「×」で閉じられます。Officeアプリでは自由に文字入力・編集ができ、ブラウザの自動保存機能が利用できます。

6. **ログイン情報**
   ログイン画面では、事前に登録したユーザー名とパスワードで認証を行います。
   （例：ユーザー名：`user` / パスワード：`user`）

### 10. 実行手順・検証方法
#### システム起動
1. Eclipseでプロジェクトを開き、`SlimeosFinalApplication.java` を実行しサーバーを起動
2. ブラウザで `http://localhost:8080/start.html` にアクセス

#### 認証・機能検証
1. ログイン画面でユーザー名・パスワードを入力し認証を実施
2. ブラウザの自動保存機能が動作することを確認し、ユーザー体験とセキュリティを検証
3. H2コンソール `http://localhost:8080/h2-console` にアクセスし、`USERS` テーブルが存在し、データが正しく保存されていることを確認
